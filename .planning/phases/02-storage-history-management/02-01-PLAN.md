2. Create `src-tauri/src/db.rs` with the following structure:

   **Database initialization:**
   - `pub fn init_db(app: &tauri::AppHandle) -> Result<(), Box<dyn std::error::Error>>` — creates SQLite file at the Tauri app data directory (use `app.path().app_data_dir()`), stores connection as a managed state via `app.manage()`. Create table `clipboard_items` with columns matching ClipboardItem struct:
     - `id TEXT PRIMARY KEY` (UUID string)
     - `content_type TEXT NOT NULL` ("text", "image", "file-paths")
     - `content TEXT NOT NULL` (actual content)
     - `hash TEXT NOT NULL UNIQUE` (SHA256 for dedup)
     - `timestamp INTEGER NOT NULL` (millisecond epoch)
     - `metadata TEXT NOT NULL DEFAULT '{}'` (JSON string)
     - `pinned INTEGER NOT NULL DEFAULT 0` (0=unpinned, 1=pinned)
     - `pinned_at INTEGER` (millisecond epoch, NULL if unpinned)
   - Create FTS5 virtual table `clipboard_items_fts` on `content` column with `content='clipboard_items'` and `content_rowid='rowid'`.
   - Create index `idx_clipboard_items_timestamp` on `timestamp DESC`.
   - Create index `idx_clipboard_items_pinned` on `pinned, pinned_at DESC`.
   - Use a `Mutex<rusqlite::Connection>` wrapped in a struct `DbState` managed by Tauri's state.

   **Insert (called from clipboard monitoring):**
   - `pub fn insert_item(conn: &Mutex<rusqlite::Connection>, item: &ClipboardItem) -> Result<(), String>` — INSERT OR IGNORE (hash unique constraint handles dedup). Also insert into FTS5 table.

   **Query history with pagination:**
   - `#[tauri::command] pub fn get_history(state: tauri::State<'_, DbState>, offset: u32, limit: u32) -> Result<Vec<ClipboardItem>, String>` — Returns items sorted by: pinned items first (by pinned_at DESC), then unpinned (by timestamp DESC). Limit default 50, offset for pagination. Deserialize metadata JSON string back to serde_json::Value.

   **Search with FTS5:**
   - `#[tauri::command] pub fn search_history(state: tauri::State<'_, DbState>, query: String, offset: u32, limit: u32) -> Result<Vec<ClipboardItem>, String>` — Uses `SELECT * FROM clipboard_items WHERE id IN (SELECT rowid FROM clipboard_items_fts WHERE clipboard_items_fts MATCH ? ORDER BY rank) ORDER BY pinned DESC, pinned_at DESC, timestamp DESC LIMIT ? OFFSET ?`. Sanitize query: if empty or whitespace, return empty array. Use FTS5 basic token query (wrap user input in double quotes to treat as phrase).

   **Delete single or batch:**
   - `#[tauri::command] pub fn delete_items(state: tauri::State<'_, DbState>, ids: Vec<String>) -> Result<(), String>` — Delete by IDs from both main table and FTS table (FTS uses `DELETE FROM clipboard_items_fts WHERE rowid IN (SELECT rowid FROM clipboard_items WHERE id = ?)` for each ID).

   **Pin/unpin:**
   - `#[tauri::command] pub fn pin_item(state: tauri::State<'_, DbState>, id: String) -> Result<(), String>` — SET pinned=1, pinned_at=now
   - `#[tauri::command] pub fn unpin_item(state: tauri::State<'_, DbState>, id: String) -> Result<(), String>` — SET pinned=0, pinned_at=NULL

   **Get pinned items:**
   - `#[tauri::command] pub fn get_pinned(state: tauri::State<'_, DbState>) -> Result<Vec<ClipboardItem>, String>` — Returns all pinned items ordered by pinned_at DESC.

   **Struct definition:**
   ```rust
   pub struct DbState {
       pub conn: Mutex<rusqlite::Connection>,
   }
   ```

   **Important implementation details:**
   - All Tauri commands must derive or use `tauri::State` properly.
   - Use `impl From<rusqlite::Error> for String` or `.map_err(|e| e.to_string())` for error handling.
   - The `ClipboardItem` struct already exists in `clipboard.rs` — import it with `use crate::clipboard::ClipboardItem;`. Add `pinned` and `pinned_at` fields to it (as `Option<i64>` for pinned_at, default None).
   - For FTS5 external content mode, after initial table creation the FTS index stays in sync via triggers or manual insert/delete — use triggers: `CREATE TRIGGER IF NOT EXISTS clipboard_items_ai AFTER INSERT ON clipboard_items BEGIN INSERT INTO clipboard_items_fts(rowid, content) VALUES (new.rowid, new.content); END;` and similar for DELETE.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend -> Tauri Commands | Untrusted input from webview crosses into Rust backend via invoke |
| Clipboard Monitor -> SQLite | System clipboard data (potentially malicious content) written to local DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | S (Spoofing) | delete_items command | mitigate | Validate ID format (UUID) before executing DELETE queries |
| T-02-02 | I (Information Disclosure) | SQLite database file | accept | Database stored in app data directory, OS-level file permissions apply; no PII in clipboard data beyond what user copies |
| T-02-03 | D (DoS) | search_history command | mitigate | Limit query string length to 200 chars, limit result count to 100, validate offset/limit bounds |
| T-02-04 | T (Tampering) | FTS5 query input | mitigate | Sanitize search query — wrap in double quotes for FTS5 phrase matching, reject special FTS5 syntax characters |
| T-02-05 | E (Elevation) | SQL injection | mitigate | Use parameterized queries exclusively (rusqlite prepared statements), never concatenate user input into SQL |
</threat_model>
