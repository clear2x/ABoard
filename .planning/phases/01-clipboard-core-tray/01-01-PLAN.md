## Key Tauri v2 API Patterns

### Clipboard Manager Plugin (Rust)
```rust
use tauri_plugin_clipboard_manager::ClipboardExt;
let content = app.clipboard().read_text();
app.clipboard().write_text("text".to_string());
```
Cargo.toml: `tauri-plugin-clipboard-manager = "2"`
Main.rs: `.plugin(tauri_plugin_clipboard_manager::init())`

### Tauri Event Emission (Rust -> Frontend)
```rust
use tauri::Emitter;
app.emit("event-name", &payload)?;
```

### Window Close Prevention (Frontend)
```typescript
import { getCurrentWindow } from "@tauri-apps/api/window";
await getCurrentWindow().onCloseRequested(async (event) => {
  event.preventDefault();
  await getCurrentWindow().hide();
});
```

### Tauri v2 Permission System
Permissions go in `src-tauri/capabilities/default.json`:
```json
{
  "identifier": "default",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "clipboard-manager:allow-read-text",
    "clipboard-manager:allow-write-text"
  ]
}
```

1. **Create `src-tauri/src/clipboard.rs`** with the clipboard monitoring module:

   Define the `ClipboardItem` struct (per CONTEXT.md decision):
   ```rust
   use serde::{Deserialize, Serialize};

   #[derive(Debug, Clone, Serialize, Deserialize)]
   #[serde(tag = "type", content = "content")]
   pub enum ClipboardContent {
       Text(String),
       Image(Vec<u8>),
       FilePaths(Vec<String>),
   }

   #[derive(Debug, Clone, Serialize, Deserialize)]
   pub struct ClipboardItem {
       pub id: String,
       #[serde(rename = "type")]
       pub content_type: String,  // "text" | "image" | "file-paths"
       pub content: String,       // For text: the text, for image: base64, for file-paths: JSON array string
       pub hash: String,
       pub timestamp: i64,
       pub metadata: serde_json::Value,
   }
   ```

2. **Implement SHA256 hashing** for deduplication:
   ```rust
   use sha2::{Sha256, Digest};

   pub fn compute_hash(content: &[u8]) -> String {
       let mut hasher = Sha256::new();
       hasher.update(content);
       format!("{:x}", hasher.finalize())
   }
   ```

3. **Implement clipboard change detection** using polling with 200ms interval (per CONTEXT.md: "event-driven with polling fallback"). For Phase 1 on macOS, use polling since Tauri clipboard-manager plugin does not have a built-in change listener. The approach:
   - Store the last known hash
   - Every 200ms, read clipboard content and compute hash
   - If hash changed, create a `ClipboardItem` and emit Tauri event

   Use `tauri::async_runtime::spawn` for the monitoring loop so it does not block the main thread.

   The monitor function signature:
   ```rust
   pub fn start_monitoring(app: tauri::AppHandle) {
       tauri::async_runtime::spawn(async move {
           let mut last_hash = String::new();
           loop {
               tokio::time::sleep(std::time::Duration::from_millis(200)).await;
               // Read clipboard via app.clipboard().read_text()
               // Compute hash, compare with last_hash
               // If different: create ClipboardItem, emit "clipboard-update" event
               // Handle errors silently (clipboard may be empty or contain unsupported types)
           }
       });
   }
   ```

4. **Handle three content types** in the polling loop:
   - Try `read_text()` first. If it returns `Ok(Some(text))` and text is not empty -> type "text"
   - If text fails or is empty, try reading image data. For Phase 1, if clipboard has image data (check via arboard or the plugin's read_image), convert to base64 -> type "image"
   - If the text looks like file paths (starts with "file://" or multiple lines of paths), parse as file paths -> type "file-paths"
   - IMPORTANT: For Phase 1, focus on text monitoring as the primary path. Image and file-paths detection should be attempted but if the plugin does not support `read_image` directly, log a debug message and skip (do not crash).

5. **Wire the monitor into `src-tauri/src/main.rs`**:
   ```rust
   mod clipboard;

   fn main() {
       tauri::Builder::default()
           .plugin(tauri_plugin_clipboard_manager::init())
           .plugin(tauri_plugin_global_shortcut::init())
           .setup(|app| {
               let app_handle = app.handle().clone();
               clipboard::start_monitoring(app_handle);
               Ok(())
           })
           .run(tauri::generate_context!())
           .expect("error while running tauri application");
   }
   ```

6. **Emit events** using `app.emit("clipboard-update", &clipboard_item)` so the frontend can listen.

7. **Add a Tauri command for pausing/resuming monitoring** (needed for tray menu "Pause Monitoring"):

   ```rust
   use std::sync::atomic::{AtomicBool, Ordering};
   static MONITORING_PAUSED: AtomicBool = AtomicBool::new(false);

   #[tauri::command]
   fn toggle_monitoring() -> bool {
       let current = MONITORING_PAUSED.load(Ordering::SeqCst);
       MONITORING_PAUSED.store(!current, Ordering::SeqCst);
       !current  // returns new state: true = paused
   }
   ```
   In the monitoring loop, check `MONITORING_PAUSED` before processing. If paused, skip the read/hash/emit cycle.
