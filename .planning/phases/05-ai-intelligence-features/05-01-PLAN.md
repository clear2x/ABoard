1. **扩展数据库 schema** (db.rs):
   - 在 `init_db` 的 `CREATE TABLE clipboard_items` 中添加新字段:
     ```sql
     ai_type TEXT,           -- AI 检测的内容类型: code, link, json, xml, image, text
     ai_tags TEXT,           -- AI 生成的标签, JSON 数组字符串, 如 ["javascript","function","async"]
     ai_summary TEXT,        -- AI 生成的摘要 (Plan 02 使用)
     ```
   - 添加 `ALTER TABLE` 迁移逻辑: 检查字段是否存在，不存在则 ALTER TABLE ADD COLUMN（兼容已有数据库）
   - 更新 `row_to_item` 函数读取新字段

2. **创建自动处理模块** (src-tauri/src/ai/processor.rs):
   - 定义 `ProcessingJob { item_id: String, content: String }` 结构体
   - 创建 `AiProcessor` struct 持有 `tokio::sync::mpsc::Sender<ProcessingJob>` 和 `app_handle`
   - 实现 `start_processor(app_handle)` -- 启动后台 task 循环接收 MPSC 消息，串行处理每条记录
   - 处理流程:
     a. **混合类型检测** (先规则，后 AI):
        - `detect_type_rules(content)` 纯正则/规则函数:
          - JSON: 尝试 `serde_json::from_str` 成功且非纯数字/字符串
          - XML: 以 `<?xml` 或 `<` 开头且包含匹配的 `</` 闭合标签
          - 链接: 正则匹配 `https?://\S+` 且内容主体就是一个 URL
          - 代码: 检测常见编程语言关键字模式 (function/def/class/import/const/let/var + 大括号/缩进等)
          - 图片: content_type 已经是 "image"（clipboard.rs 中检测）
        - 如果规则检测成功 -> 直接使用结果
        - 如果规则不确定 -> 构建 InferenceRequest 调用 AI，prompt: "分析以下内容并返回其类型。只能返回: code, link, text, json, xml。内容:\n{content}"
     b. **语义标签生成**:
        - 构建 prompt: "为以下{type}内容生成 3-5 个语义标签，用 JSON 数组格式返回。内容:\n{content}"
        - 调用 ai_infer 推理（使用现有 AiState provider）
        - 解析返回的 JSON 数组
     c. **更新数据库**: 调用 `update_ai_metadata` 写入 ai_type 和 ai_tags
     d. **发送 Tauri event**: emit "ai-processed" event 到前端，携带 { item_id, ai_type, ai_tags }

3. **集成到剪贴板监听** (clipboard.rs):
   - 在 `start_monitoring` 中获取 `AiProcessor` 引用
   - 在捕获新内容后（insert_item 之后），发送 ProcessingJob 到队列
   - 使用 `app.state::<AiProcessor>()` 获取 processor 引用

4. **注册新 commands** (lib.rs):
   - 在 `invoke_handler` 中注册 `ai::processor::update_ai_metadata`
   - 在 `setup` 中调用 `ai::processor::start_processor(&app.handle())`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| clipboard -> processor | 剪贴板内容进入处理队列，可能包含恶意构造文本 |
| processor -> AI inference | AI 推理请求包含用户剪贴内容，需防止 prompt injection |
| AI response -> database | AI 返回的标签/类型写入数据库，需验证格式 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | I (Information Disclosure) | processor -> AI API | accept | AI 推理请求携带用户剪贴内容，仅在本地模型或用户自配置 API 下发送 |
| T-05-02 | D (Denial of Service) | MPSC channel | mitigate | 设置 channel 容量上限(100)，超出丢弃并记录日志 |
| T-05-03 | T (Tampering) | AI response -> DB | mitigate | 验证 AI 返回的 ai_type 在允许枚举范围内，ai_tags 为合法 JSON 数组且每项长度<=50 |
| T-05-04 | S (Spoofing) | Tauri event | accept | 前端仅更新本地状态，无安全影响 |
</threat_model>
