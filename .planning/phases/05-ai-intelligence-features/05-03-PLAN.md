1. **创建 AI 操作 store** (src/stores/ai-actions.ts):

   定义结果状态接口:
   ```typescript
   export interface AiActionResult {
     originalContent: string;
     resultText: string;
     actionType: "translate" | "summarize" | "rewrite";
     itemId: string;
   }

   // 结果弹窗状态
   const [resultPopup, setResultPopup] = createSignal<AiActionResult | null>(null);
   const [processing, setProcessing] = createSignal<string | null>(null); // "translate" | "summarize" | "rewrite" | null
   export { resultPopup, setResultPopup, processing, setProcessing };
   ```

   实现三个操作函数，均调用 `invoke("ai_infer_auto", { request })`:

   a. `translateContent(content: string, itemId: string)`:
      - 自动检测语言方向: 检查内容是否包含中文字符 (`/[\u4e00-\u9fff]/` 测试)
      - 中文 -> 英文: system_prompt = "你是一个翻译助手。将以下中文翻译为自然流畅的英文。只返回翻译结果。"
      - 非中文 -> 中文: system_prompt = "你是一个翻译助手。将以下内容翻译为自然流畅的中文。只返回翻译结果。"
      - max_tokens: 根据 content 长度动态设置 (Math.max(content.length * 2, 500))

   b. `summarizeContent(content: string, itemId: string)`:
      - system_prompt = "你是一个总结助手。将以下内容总结为要点列表，每条以数字编号。保持简洁准确。"
      - max_tokens: 500

   c. `rewriteContent(content: string, itemId: string, style: string)`:
      - style 映射:
        ```
        "formal" -> "正式风格：使用书面语言，避免口语化表达"
        "casual" -> "随意风格：使用轻松口语化表达，像朋友聊天"
        "concise" -> "简洁风格：去除冗余，用最少的文字传达核心意思"
        "detailed" -> "详细风格：展开描述，增加细节和解释"
        "academic" -> "学术风格：使用学术语言，严谨的逻辑和术语"
        ```
      - system_prompt = `你是一个改写助手。请用${styleDesc}改写以下内容。保持核心意思不变。只返回改写结果。`

   每个函数:
   - 设置 `processing` signal 表示正在处理
   - 调用 `invoke("ai_infer_auto", { request: { prompt: content, system_prompt, max_tokens, temperature: 0.7 } })`
   - 成功后设置 `setResultPopup({ originalContent, resultText: response.response.text, actionType, itemId })`
   - 失败时 console.error + 设置 processing 为 null
   - finally 清除 processing

2. **创建 AiResultPopup 组件** (src/components/AiResultPopup.tsx):
   - 导入 resultPopup, setResultPopup from ai-actions store
   - 导入 invoke from @tauri-apps/api/core
   - 导入 addItem from stores/clipboard
   - Props: 无（使用 store signal 驱动）
   - 当 resultPopup() 不为 null 时渲染弹窗
   - 弹窗结构:
     ```tsx
     <Show when={resultPopup()}>
       <div class="fixed inset-0 z-50 flex items-center justify-center">
         {/* 遮罩 */}
         <div class="absolute inset-0 bg-black/50 backdrop-blur-sm" onClick={close} />
         {/* 弹窗主体 */}
         <div class="glass-card relative z-10 w-[500px] max-h-[70vh] p-4 rounded-[var(--radius-lg)]"
              style={{ background: "var(--glass-bg)", border: "1px solid var(--glass-border)", "box-shadow": "var(--shadow-elevated)" }}>
           {/* 标题 */}
           <h3>{actionTitle}</h3>
           {/* 原文区域 */}
           <div class="mb-2 p-2 rounded bg-white/5 text-xs text-[var(--color-text-muted)]">
             {result().originalContent}
           </div>
           {/* 结果区域 */}
           <div class="p-3 rounded bg-white/5 max-h-[40vh] overflow-y-auto text-sm"
                style={{ color: "var(--color-text-secondary)" }}>
             {result().resultText}
           </div>
           {/* 操作按钮 */}
           <div class="flex gap-2 mt-3">
             <button onClick={handleCopy}>复制结果</button>
             <button onClick={handleReplace}>替换原内容</button>
             <button onClick={handleAppend}>追加为新条目</button>
           </div>
         </div>
       </div>
     </Show>
     ```
   - handleCopy: 使用 `navigator.clipboard.writeText(result().resultText)`
   - handleReplace: 调用 Rust command 更新数据库中对应条目的 content
   - handleAppend: 创建新 ClipboardItem 并通过 addItem 添加到列表（同时写入数据库）
   - 弹窗动效: 使用 CSS animate 类实现淡入缩放 (参考现有 animate-context-menu)
   - ESC 键关闭: 监听 keydown 事件
