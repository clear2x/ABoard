在 Plan 01 创建的 processor.rs 处理循环中添加摘要生成步骤:

1. 在类型检测和标签生成之后，检查内容长度:
   - 计算 `content.chars().count()`
   - 仅当字符数 > 200 时触发摘要生成 (per CONTEXT.md D-01)
   - 短文本直接跳过，ai_summary 保持 NULL

2. 摘要生成 prompt 设计:
   ```rust
   let summary_prompt = format!(
       "用 1-2 句话总结以下内容的要点，要求简洁准确，不超过 100 字：\n\n{}",
       &content[..content.len().min(2000)]  // 截断过长内容避免超 token
   );
   let request = InferenceRequest {
       prompt: summary_prompt,
       system_prompt: Some("你是一个文本摘要助手。只返回摘要文本，不要添加前缀或解释。".to_string()),
       max_tokens: Some(150),
       temperature: Some(0.3),  // 低温度保证摘要质量稳定
       top_p: None,
   };
   ```

3. 调用 `ai_infer_auto` 获取摘要结果

4. 更新数据库: 在同一个 `update_ai_metadata` 调用中写入 `ai_summary`

5. 更新 "ai-processed" event payload 添加 `ai_summary` 字段:
   ```json
   { "item_id": "...", "ai_type": "code", "ai_tags": ["..."], "ai_summary": "..." }
   ```

6. 错误处理: 摘要生成失败不影响类型和标签的保存 -- 每个步骤独立 catch
