重写 src-tauri/src/ai/local.rs，将 stub 替换为实际推理实现。

**策略**: 使用 HTTP 调用本地推理服务（Ollama API 或 llama.cpp server）。这是最实际的方案：
1. Ollama 是本地推理的事实标准，自带 OpenAI 兼容 HTTP API (localhost:11434)
2. llama.cpp server 也提供 HTTP API (默认 localhost:8080)
3. 无需链接任何 C/C++ 原生库，保持 Cargo 构建简洁
4. 用户只需安装 Ollama（一个命令即可），或启动 llama.cpp server

**LocalProvider 结构体改造**:
```rust
pub struct LocalProvider {
    model_path: String,
    client: reqwest::Client,
    ollama_endpoint: String,     // 默认 "http://localhost:11434"
    llamacpp_endpoint: String,   // 默认 "http://localhost:8080"
}
```

**is_available() 实现**:
- 向 ollama_endpoint 发送 GET /api/tags，如果 200 且返回模型列表 → Ollama 可用
- 否则向 llamacpp_endpoint 发送 GET /health，如果 200 → llama.cpp server 可用
- 缓存检测结果（使用 std::sync::atomic::AtomicBool + last_check 时间戳，避免每次调用都检测）
- 任一服务可用即返回 true

**infer() 实现**:
- 优先尝试 Ollama API: POST http://localhost:11434/api/chat
  - Body: { model: 从 model_path 提取模型名或使用默认 "llama3.2", messages: [...], stream: false, options: { temperature, top_p, num_predict } }
  - 解析 response: json["message"]["content"] 提取文本
- 如果 Ollama 不可用，尝试 llama.cpp server: POST http://localhost:8080/v1/chat/completions
  - Body: 标准 OpenAI 格式 { messages: [...], max_tokens, temperature, top_p }
  - 解析 response: json["choices"][0]["message"]["content"]
- 两者都不可用时返回明确错误信息："No local inference service found. Install Ollama (https://ollama.com) or start llama.cpp server."

**构造函数改造**:
- new(model_path) 保持签名兼容
- 内部创建 reqwest::Client、设置默认 endpoint

**不需要修改 Cargo.toml**: reqwest 已有 json feature，tokio 异步运行时已就绪。

关键约束:
- 保持 LocalProvider::new(String) 的公共签名不变（mod.rs init_ai 和 ai_set_provider 都使用它）
- 保持 InferenceProvider trait impl 不变
- 不要引入新的 Rust crate 依赖
- 错误信息要用户友好，包含安装指引
