   - `reqwest = { version = "0.12", features = ["json"] }` （云端 HTTP 客户端）
   - 注意 reqwest 需要 tokio runtime，项目已有 tokio 依赖

2. 创建 src-tauri/src/ai/mod.rs:
   - 定义 `InferenceProvider` trait:
     ```rust
     #[async_trait::async_runtime::async_trait(?Send)]
     pub trait InferenceProvider: Send + Sync {
         async fn infer(&self, request: InferenceRequest) -> Result<InferenceResponse, String>;
         fn name(&self) -> &str;
         fn is_available(&self) -> bool;
     }
     ```
   - 定义数据结构:
     ```rust
     #[derive(Debug, Clone, Serialize, Deserialize)]
     pub struct InferenceRequest {
         pub prompt: String,
         pub system_prompt: Option<String>,
         pub max_tokens: Option<u32>,      // 默认 512
         pub temperature: Option<f32>,     // 默认 0.7
         pub top_p: Option<f32>,           // 默认 0.9
     }
     
     #[derive(Debug, Clone, Serialize, Deserialize)]
     pub struct InferenceResponse {
         pub text: String,
         pub tokens_used: u32,
         pub provider: String,             // "local" | "openai" | "anthropic"
         pub duration_ms: u64,
     }
     ```
   - 定义 `AiState` 管理 struct，持有当前活跃 provider（Arc<Mutex<Box<dyn InferenceProvider>>>）和配置
   - pub mod local; pub mod cloud; pub mod config;
   - 提供 `init_ai(app: &tauri::AppHandle)` 函数创建默认 AiState

3. 创建 src-tauri/src/ai/config.rs:
   - `ProviderType` 枚举: Local, OpenAi, Anthropic
   - `AiConfig` struct:
     ```rust
     pub struct AiConfig {
         pub active_provider: ProviderType,
         // Local
         pub model_path: Option<String>,
         pub context_length: u32,          // 默认 2048
         // OpenAI compatible
         pub openai_api_key: Option<String>,
         pub openai_endpoint: String,      // 默认 "https://api.openai.com/v1"
         pub openai_model: String,         // 默认 "gpt-4o-mini"
         // Anthropic
         pub anthropic_api_key: Option<String>,
         pub anthropic_model: String,      // 默认 "claude-sonnet-4-20250514"
     }
     ```
   - 实现 `AiConfig::default()` 和 `AiConfig::save() / load()` 方法
   - save/load 使用 app_data_dir 下的 `ai-config.json` 文件（参考 db.rs 的 app_data_dir 获取方式）

   - `LocalProvider` struct 持有 `Option<llama_cpp_rs::LlamaModel>` 和 `model_path: String`
   - 实现 InferenceProvider trait:
     - `infer()`: 如果 model 为 None 返回 Err("No model loaded")
     - 使用 llama-cpp-rs 的 session 创建 + infer 方法
     - 将 prompt 构造为完整输入（考虑 system_prompt）
     - 捕获耗时和 token 数
     - `is_available()`: model.is_some()
     - `name()`: "local"
   - 提供 `LocalProvider::load_model(path: &str) -> Result<Self, String>` 构造方法

2. 创建 src-tauri/src/ai/cloud.rs:
   - `OpenAiProvider` struct 持有 reqwest::Client, api_key, endpoint, model_name
   - 实现 InferenceProvider:
     - POST {endpoint}/chat/completions
     - Body: { model, messages: [{role:"system",content:sys_prompt}, {role:"user",content:prompt}], max_tokens, temperature, top_p }
     - 解析 response.choices[0].message.content
     - `is_available()`: api_key.is_some()
   - `AnthropicProvider` struct 持有 reqwest::Client, api_key, model_name
   - 实现 InferenceProvider:
     - POST https://api.anthropic.com/v1/messages
     - Headers: x-api-key, anthropic-version: "2023-06-01", content-type: application/json
     - Body: { model, max_tokens, system: sys_prompt, messages: [{role:"user",content:prompt}] }
     - 解析 response.content[0].text
     - `is_available()`: api_key.is_some()

3. 修改 src-tauri/src/lib.rs:
   - 添加 `mod ai;`
   - 在 setup 闭包中调用 `ai::init_ai(&app.handle())?;`（在 db::init_db 之后）

     - `ai_list_models(state) -> Result<Vec<ModelInfo>, String>`: 扫描 models 目录下 .gguf 文件（Phase 2 的 models.rs 会增强此功能）
     - `ai_set_provider(state, provider_type, config) -> Result<(), String>`: 切换活跃 provider

   - AiState 通过 `app.manage(AiState { ... })` 注册为 Tauri state

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend -> ai_infer command | 用户输入的 prompt 经由 Tauri IPC 传入 Rust |
| Rust -> External API (OpenAI/Anthropic) | API key 通过 HTTPS 传输，prompt 明文发送 |
| Config file (ai-config.json) | 存储在 app_data_dir，包含 API keys |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | I | ai_infer prompt | mitigate | 限制 prompt 最大长度 100KB，防止内存耗尽 |
| T-04-02 | S | External API calls | mitigate | API key 不返回给前端，仅在 Rust 侧使用 |
| T-04-03 | D | ai-config.json | accept | 本地桌面应用，文件系统权限由 OS 管理 |
| T-04-04 | E | ai_infer | mitigate | 单次推理超时 60s，防止挂起 |
</threat_model>
